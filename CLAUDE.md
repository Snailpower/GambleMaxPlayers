# GambleMaxPlayers — BepInEx Mod

## Project overview
BepInEx 5 mod for **Gamble With Your Friends** (Steam) that raises the maximum lobby size beyond the default of 6.

## Game details
- **Install path:** default Steam location — defined locally in `GameDir.props` (gitignored)
- **Engine:** Unity (netstandard2.1 assemblies)
- **Networking:** Mirror + FizzySteamworks transport (Steam relay)
- **BepInEx version:** 5.4.23, already installed in the game directory

## Key game classes
- **`LobbySettings`** (`Assembly-CSharp.dll`) — ScriptableObject loaded via `Resources.Load<LobbySettings>("LobbySettings")`. Has a public `int maxPlayers` field that is the single source of truth for lobby size.
- **`LobbyManager`** — Mono singleton. `CreateNewLobby()` calls `SteamMatchmaking.CreateLobby(ELobbyType, lobbySettings.maxPlayers)`. Holds `lobbySettings` as a private field.
- **`CustomNetworkManager`** — Extends Mirror's `NetworkManager`. Does not override `maxConnections`; it comes from the serialized prefab (default 6). Mirror uses this in `NetworkServer.Listen(maxConnections)` on server start.

## How the mod works
Two mechanisms, split across `Plugin.cs` and `Patches.cs`:

1. **`Plugin.Awake()` — direct resource mutation** — calls `Resources.Load<LobbySettings>("LobbySettings")` and sets `maxPlayers` to the config value. Unity's resource cache means all subsequent reads (including `LobbyManager.CreateNewLobby`) see the updated value without any Harmony patch.

2. **`NetworkServerListenPatch` in `Patches.cs`** — Harmony prefix on `NetworkServer.Listen(ref int maxConns)`. Overwrites the `maxConns` ref argument with the config value so Mirror enforces the correct connection limit regardless of which NetworkManager subclass started the server.

Config file (auto-generated by BepInEx on first run):
`BepInEx/config/com.gamblewithfriends.maxplayers.cfg` — key `MaxPlayers`, default `10`, range `2–100`.

## Build & deploy
```
dotnet build                # output → build\GambleMaxPlayers.dll
deploy.bat                  # builds + copies DLL to the active BepInEx plugins folder
package_thunderstore.bat    # builds + creates GambleMaxPlayers-{version}.zip for Thunderstore upload
```

Output always lands in `build\GambleMaxPlayers.dll` regardless of configuration — `AppendTargetFrameworkToOutputPath` and `AppendRuntimeIdentifierToOutputPath` are both disabled.

BepInEx is managed via **Thunderstore Mod Manager** (not installed directly in the Steam game folder). The deploy target is the mod manager's profile plugin subfolder:
```
%APPDATA%\Thunderstore Mod Manager\DataFolder\GambleWithYourFriends\profiles\Default\BepInEx\plugins\Snailpower-GambleMaxPlayers\
```
`GameDir.props` overrides `BepInExDir` to point at the profile's `BepInEx\core\` for building, while `ManagedDir` still resolves from the Steam game folder. The mod manager syncs the plugins folder on each game launch — deploy before launching or the DLL will persist since it lives in the mod's own subfolder (`Snailpower-GambleMaxPlayers\`).

## Thunderstore packaging
Staging files live in `thunderstore/`:
- `manifest.json` — package metadata (name, version, description, BepInEx dependency)
- `icon.png` — 256×256 PNG icon (replace with a proper design before uploading)

`package_thunderstore.bat` builds the project, copies the DLL into `thunderstore/BepInEx/plugins/`, copies `README.md`, and zips everything into `GambleMaxPlayers-{version}.zip` ready for upload.

To update the version: bump `PluginVersion` in `Plugin.cs` and `version_number` in `thunderstore/manifest.json` together.
